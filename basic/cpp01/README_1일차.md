## 1. 절차지향 VS 객체지향

| 구분 | 절차지향 (Procedural) | 객체지향 (Object-Oriented) |
|---|---|---|
| **대표 언어** | C | Java, C++, Python |
| **실행 속도** | 빠름 | 상대적으로 느림 |
| **중심 개념** | 함수(Function) | 클래스(Class) |
| **처리 방식** | 순차적 처리 | 객체 단위 처리 |
| **특징** | 절차 중심 | 역할 분담에 유리 |

## 2. 구조체와 클래스

#### 구조체
- `필드` : 구조체 안의 요소
- 필드에 들어갈 수 있는것은 `변수` 뿐
    > 멤버 변수
- C++에서는 구조체에 멤버 변수와 함수를 집어 넣을 수 있다
    > Java는 불가능
- 구조체를 사용하기 위해선 구조체 변수를 선언

#### 클래스
- C언어의 `구조체에서 발전`된 형태
- 필드 안에 함수를 집어 넣을 수 없을까? >> `클래스`
- 클래스를 사용하기 위해선 클래스에 맞는 `객체`(변수)를 선언

### 살펴보기

> int a ;

- 메모리공간에 int크기의 a라는 변수를 할당
- 메모리에 접근하기 위해서는 주소가 필요
- 주소대신 a라는 변수 이름으로 접근을 하는 것

> a = 10;

- a라는 메모리 공간에 10을 집어넣음

> int* pa;
- 포인터 변수 선언, 기능이 없고 나는 포인터 변수다 하는 기호일 뿐

> pa = &a;

> *pa = value;    // 여기에 *은 연산자

#### 포인터 연산자

> `*` : 간접 참조 연산자

> `&` : 주소 연산자


#### 멤버 접근 연산자

> `->` : 화살표 연산자

> `.` : 점(도트) 연산자

a & b : 비트(논리) AND
a && b : 논리 AND

### 메모리 구조와 변수의 저장 위치
컴퓨터가 프로그램을 실행할 때 메모리는 크게 다음과 같이 나뉜다

#### 영역별 특징
> **데이터 세그먼트** : 전역 변수, static변수가 저장
- 문자 리터럴(e.g., `"hong"`) 자체가 저장되는 공간(read-only)

> **스택** : 지역 변수, 매개 변수가 저장됨
- 함수가 종료되면 자동으로 소멸, `const` 지역변수도 이 영역에 생성

> **힙** : 동적 할당(`malloc`, `new`)시 사용되는 영역

> **코드 세그먼트** : 작성한 코드(기계어)와 함수가 저장되는 곳
- 함수 이름 자체가 이 영역의 주소

#### 문자열 선언의 차이
> char arr[] = "hong";   // "hong"을 스택 영역에 복사 (수정 가능)

> char* arr1 = "hong";   // arr1은 스택에, 실제 "hong"은 데이터 세그먼트에 존재 (수정 불가)
