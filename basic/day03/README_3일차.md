### 2. 참조자(Reference) 깊게 알기
참조자는 변수 이름에 붙는 **또 다른 이름(별명)**

#### 주요특징
- **별칭 개념** : 참조자는 새로운 메모리 공간을 차지하지 않고, 기존 저장공간 자체를 참조
- `lvalue`와 `rvalue`:
    - `lvalue` : 이름이 있고 주소를 취할 수 있는 변수
    - `rvalue` : 임시적인 값(리터럴 10, 20 등).

#### 상수 참조 (Const Reference)
- `int &ref = 4;`(오류)
    - 리터럴은 변수가 아니므로 일반 참조 불가
- `const int &ref = 4;`(가능)
    - `const`를 붙이면 이름없는 상수를 참조할 수 있게 됨
- `int &&ref = 4;`(가능)
    - C++11의 **rvalue 참조**를 이용하면 리터럴 참조가 가능

#### 메모리 수명과 참조
- **스택 영역** : 지역변수가 저장되는 곳
>   함수가 `return`을 호출하고 괄호`}`가 닫히면 이 공간은 소멸

- 위험한 참조 : 함수 내부의 지역 변수를 참조로 리턴하면, 함수가 종료된 후 사라진 메모리를 가리키게되어 오류가 발생

### 3. 동적할당 : `new` & `delete`
`malloc`과 `free`를 대체하는 C++의 방법
#### 특징
- 바이트 단위를 직접 계산할 필요가 없음
- **생성자**를 자동으로 호출하여 객체를 생성

#### 힙에 포인터 없이 접근
```
int *ptr = new int;   // 힙 영역에 메모리 할당
int &ref = *ptr;      // 할당된 공간에 'ref'라는 별명을 붙임
ref = 20;             // 포인터를 쓰지 않고도 힙 메모리 값 변경 가능
cout << *ptr << endl; // 20 출력
delete ptr;           // 힙 메모리는 반드시 수동 소멸 필요
```

### 5. 클래스(Class)의 기본
클래스는 **확장된 구조체**의 형태

#### 접근 제어 지시자(Access Specifiers)
|지시자|설명|특징|
|---|---|---|
|**public**|어디서든 접근 허용|주로 멤버함수(기능)|
|**protected**|상속 관계의 클래스까지 허용|상속 시 사용|
|**private**|클래스 내부에서만 접근 가능|주로 멤버 변수(데이터), (데이터 보호)|

#### 관례 : 멤버 변수는 `private` 로 숨기고, 이를 조작하는 함수는 `public`으로 공개
#### 용어 정리
- **객체** : 클래스를 틀로 삼아 생성된 실체(변수)
- **멤버 변수** : 클래스 내부의 변수
- **멤버 함수** : 클래스 내부의 함수

### 7. name = "abcd"가 안되는 근본적인 이유
#### 배열의 이름은 `주소값 상수`
- `char name[20]`이라고 선언하는 순간, `name`은 메모리의 특정 위치를 가리키는 고정된 이정표가 됨
- `name = "abcd"`라고 쓰는 것은 "이정표 자체를 뽑아서 다른 곳에 심으라"는 명령
#### 해결책
- 땅(메모리)은 그대로 두고, 그 위에 적힌 내용물만 갈아치우는 `strcpy`를 사용

### 8. 객체 생성 문법 : `()` vs `=`
C++에서 객체를 만들 때 두 가지 방식이 모두 작동하는 이유
```
Human h("hong", 30, "학생");   // 1. 직접 초기화 (Direct Initialization)
Human h = {"hong", 30, "학생"}; // 2. 복사 초기화 (Copy Initialization)
```
#### 왜 `=`을 써도 될까?
- 대입이 아닌 `초기화`
- 컴파일러는 `=`을 보는 순간, "오른쪽에 있는 데이터들을 가지고 Human 객체를 하나 임시로 만든 다음, 그걸 `h`에 복사해서 넣어줘야지!"라고 해석
> 이를 **복사 초기화**라고 함
- 최근의 컴파일러는 똑똑해져서 거의 똑같이 처리해주기도 함

---
### 9. 객체지향에서 '복사'를 지양하는 이유
#### 1) 불필요한 오버헤드(성능 저하)
- `Human h = {. . .}` 방식을 쓰면 이론적으로
> **임시 객체 생성 -> 복사 -> 임시 객체 소멸**의 3단계를 거침 

- 데이터가 작을땐 ㄱㅊㄱㅊ, 근데 커지면 복사만해도 느려짐

#### 2) 얕은 복사의 재앙
- 만약 클래스 내부에서 `new`를 통해 메모리를 동적으로 할당 받고 있다면
> 객체를 복사할때 **주소값만 복사**되는 문제가 생김

- **결과** 
    - 두 객체가 하나의 메모리를 가리키고
    - 한쪽이 소멸될 때 메모리를 해제해버리면 다른한쪽이 `사라진 메모리`를 붙잡고 있다가 프로그램이 뻗어버림
    > **런타임 에러**의 주범

### 2. 구조체(struct) vs 클래스(class)
C++에서 구조체와 클래스는 거의 같지만, **접근 제어**의 기본값이 다름
#### 구조체(`struct`)
- 기본이 `public`(열려있음)
#### 클래스(`class`)
- 기본이 `private`(닫혀 있음 - 데이터 보호)

### 5. 클래스의 선언과 구현 분리
코드가 길어지면 설계도(선언)와 실제 동작(구현)을 나눔
#### 범위 지정 연산자 ( `::` )
클랫 외부에서 함수를 정의할 때는 반드시 **"이 함수는 어느 클래스 소속인가"**를 밝혀야 함
```
// [반환타입] [클래스이름]::[함수이름]
void MyClass::get_data() {
    // 구현 내용
}
```

### 6. 생성자 심화 : [초기화 리스트와 오버로딩](./basic/day03/init02/init02.cpp)
#### 생성자 오버로딩
매개변수의 개수나 타입에 따라 여려 개의 생성자를 둘 수 있음
- **기본 생성자(`Time()`)** : 인자 없이 `Time now;`라고 선언할 때 호출
- 매개변수가 있는 생성자(`Time(int g, int m, int s)`) : 객체 생성과 동시에 특정 값으로 초기화

#### 콜론 초기화(Member Initializer List)
C++에서 가장 권장되는 멤버 변수 초기화 방식
```
// 대입 연산이 아니라 '초기화' 그 자체를 수행
Time(int h, int m, int s) : m_hour(h), m_min(m), m_sec(s) { }
```
#### 왜쓰는가?
- 생성자 몸체( `{ }` )에서 값을 대입하는 것보다 **성능상 이점**이 있고, 특히 `const`멤버 변수나 `참조자`멤버는 오직 이 방식으로만 초기화가 가능하기 때문

#### 멤버 변수 명명 관례 (`m_`)
- 멤버 변수 앞에 **`m_`**을 붙이는 것은 이 변수가 클래스의 **Member(필드)**임을 명시하는 관례
- **장점** : 매개변수(`h`, `m`, `s`)와 멤버 변수가 섞일 염려가 없고 코드의 가독성을 높여줌

### 7. 초기화 리스트(Initializer List)가 필수인 경우 : [소스](./basic/day03/init03/init03.cpp)
일반적인 멤버 변수는 생성자 몸체(`{ }`) 안에서 대입 연산자로 값을 넣을 수 있지만, 아래의 두 경우에는 반드시 **콜론(:) 초기화 리스트**를 사용
#### 1) 상수 멤버 변수(const)
- 상수는 탄생과 동시에 값이 결정되어야 하며, 이후에는 값 변경(대입) 불가
- 생성자 몸체 안에서 `value = i;`라고 쓰는 것은 '대입'이지 '초기화'가 아니므로 에러가 발생

#### 2) 참조자 멤버 변수(&)
- 참조자는 선언과 동시에 누구를 참조할 지 별명이 붙어야 함
- 참조자 역시 한 번 결정되면 참조 대상을 바꿀 수 없으므로, 생성자 몸체가 시작되기 전에 초기화 리스트에서 연결

### 8. C/C++ 객체지향 진화 및 생성자 핵심 요약

| 구분 | 핵심 특징 | 코드 예시 | 비고 |
| :--- | :--- | :--- | :--- |

| **C++ 구조체** | **함수 포함 가능** | `h.show_data();` | 데이터가 스스로 기능을 '직접 실행' |

| **구현 분리** | **가독성/관리 용이** | `void MyClass::get_data()` | `::`(범위 지정 연산자)를 사용해 외부 정의 |

| **필수 초기화** | **const, 참조자(&)** | `: value(i), ri(i)` | **상수와 참조자 멤버는 필수적으로 사용** |
