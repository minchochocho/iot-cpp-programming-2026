### 7. 입력 버퍼(Input Buffer)
#### 입력 버퍼란?
- 키보드로 입력한 데이터는 즉시 변수로 들어가지 않는다
- 먼저 `입력 버퍼`에 저장된다
- 입력 함수는 이 입력 버퍼에 저장된 데이터를 읽어 변수에 저장한다

#### 입력 과정의 흐름
1. 사용자가 키보드로 입력
2. 엔터를 누름
3. 엔터 전까지의 모든 내용이 **입력 버퍼에 저장**
4. 입력함수(`cin`, `scanf`)가 호출되면
 -> 입력 버퍼의 데이터를 읽어 변수에 저장

#### 문자열 입력과 `cin`
- `cin`은 입력 버퍼에 저장된 데이터를 읽는 개체
- 공백을 기준으로 데이터를 읽음
- `cin`이 호출되면
    -> 버퍼에 있는 문자열을 변수로 가져옴
- scanf도 마찬가지

### 1. C와 C++의 함수 구분 방식

#### C 언어

- 매개변수가 달라도

#### C++ 언어
- 함수이름 + 매개변수 목록으로 함수를 구분
- 매개변수의 개수, 타입, 순서가 모두 같아야 함
- 한환형(출력 타입은) 함수 구분에 포함 XX

| 언어 | 함수 구분 방식 | 오버로딩 |
|:------|:---------------|:--------:|
| **C** | 이름 | X |
| **C++** | 이름 + 매개변수 | O |
---
### 2. 함수 오버로딩
#### 함수 오버로딩이란?
- **같은 이름**의 함수를 여러개 정의 하는 것
- 단, 매개변수 선언이 서로 달라야 함
- 컴파일러가 호출 시 전달된 인자를 기준으로 어떤 함수를 호출할지 결정
```
int sum(int a, int b);
int sum(int a, int b, int c);
double sum(double a, double b);
```
> 위 함수들은 서로 다른 함수로 인식됨
---
### 3. 디폴트 매개변수(Default Prameter)
#### 디폴트 값이란?
- 함수의 매개변수에 기본값을 미리 설정하는 것
- 함수 호출 시 해당 인자를 전달하지 않으면
    -> 설정된 디폴트 값이 자동으로 사용
```
void print(int x = 10);

print();    // x = 10
print(5);   // x = 5
```
---
#### 부분 디폴트 값(Partial Default Parameter)
- 모든 매개변수에 디폴트 값을 줄 필요는 없음
- 일부 매개변수만 디폴트 값 설정 가능
- 이를 **부분 디폴트 값 설정**이라고 함
```
void func(int a, int b = 10);

func(3);        // a = 3, b = 10
func(3, 5);     // a = 3, b = 5
```
---
#### 디폴트 값 설정 규칙**(중요)**
- 디폴트 값은 반드시 **뒤쪽 매개변수**부터 설정해야 한다
> void func(int a, int b = 10, int c = 20);  // ⭕ 가능

> void func(int a = 10, int b);  // ❌ 불가능
---
#### 이유
- 함수 호출시 인자는 **왼쪽부터** 순서대로 매칭
- 앞 매개변수가 디폴트면 
-> 뒤 매개변수에 어떤 값을 넣어야 할지 판단 불가
---
### 4. 인라인 함수
#### 인라인 함수란?
- 함수 호출시 분기(branch)없이 함수정의가 **호출** 위치에 그대로 삽입되는 함수
- 함수 호출 코드가 **함수 정의코드**로 치환
---
#### 인라인 함수의 장점
- 함수 호출에 따른 **오버헤드 제거**
- 실행 속도가 향상될 수 있음
---
#### 인라인 함수 선언 방법
- 일반함수 앞에 `inline` 키워드를 붙이면
    - **인라인 함수**로 선언
- `inline`키워드는 컴파일러에게 힌트일 뿐
- 컴파일러의 판단에 따라
    - 선언이 있어도 처리 X
    - 선언 없이 인라인 처리
---
#### 인라인 함수의 한계(단점)
- 함수 정의 방식이 일반 함수보다 **복잡**
- 함수 내용이 길거나 복잡하면
    - 인라인 처리되지 않거나
    - 코드 크기 증가로 비효율적
- **짧고 단순**한 함수에 적합
---
### 5. 매크로 함수
#### 매크로 함수란?
- 전처리 단계에서 **문자열 치환 방식**으로 처리되는 함수 형태의 매크로
- 함수 호출 개념이 없음
> #define SQUARE(x) ((x) * (x))
---
#### 매크로 함수의 장점
- 자료형에 독립적
    - 타입 개념이 없기 때문에
    - 모든 자료형에 동일하게 적용 가능
- 오버로딩이 필요없음
    - 인라인 함수는 자료형마다 오버로딩 필요
    - 매크로 함수는 하나의 정의로 모든 타입 처리 가능
---
#### 매크로 함수의 단점
- **타입 체크 불가**
    - 컴파일러가 아닌 전처리기 단계에서 처리
- 디버깅이 어려움
    - 오류 발생 시 실제 코드 위치 파악이 힘듦
- 부작용 발생 가능
    - 인자에 식이 들어갈 경우 예상치 못한 결과 발생
    > SQUARE(a++) → ((a++) * (a++))
- 코드 가독성과 유지보수성 저하
---
### 6. 템플릿(Template)
#### 템플릿이란?
- 대부분 인라인 함수로 동작
- 자료형에 독립적인 코드 작성
- 컴파일 시 자료형별 함수/클래스 생성
- 그 결과로 인라인 처리될 가능성이 높다
---
### 7. 매크로 함수와 인라인 함수(템플릿)
| 구분 | 매크로 함수 | 인라인 함수 | 템플릿 |
|---|---|---|---|
| 처리 시점 | 전처리 단계 | 컴파일 단계 | 컴파일 단계 |
| 처리 방식 | 문자열 치환 | 호출 위치에 함수정의 | 자료형별 함수 생성 |
| 자료형 | 자료형 독립적 | 자료형 종속적 | 자료형 독립적 |
| 오버로딩 | 불가능 | 가능 | 자동 생성 |
| 타입 검사 | 불가능 | 가능 | 가능 |
| 성능 | 빠름 | 빠를 수 있음 | 빠를 수 있음 |
| 특징 | 단순·위험 | 안전·힌트 | 정적 다형성 |

> 매크로 함수: 자료형에 독립적이지만 위험

> 인라인 함수: 안전하지만 자료형별 정의 필요

### 3. 동일한 namespace 내에서의 함수 호출

### 4. namespace의 중첩
#### namespace의 중첩이란?
- 네임스페이스 안에 또 다른 네임 스페이스를 선언하는 것
- 계층적 구조로 구성 가능
```
namespace Parent {
    int num = 2;

    }

    }
}
```

#### 중첩된 이름공간 접근 방법
- `::` 연산자를 사용하여 계층적으로 접근
```
Parent::num            // 2
Parent::SubOne::num    // 3
Parent::SubTwo::num    // 4
```

    - 모두 namespace `std`안에 선언

### 6. using을 이용한 namespace의 명시

#### 1) using 선언
- 네임스페이스에 속한 **특정 식별자**만 현재 범위로 가져옴
- 이름 충돌 위험이 비교적 적음

```

using std::cout;

    int x;
    cin >> x;
    cout << x << endl;
}
```
---
#### 2) using namespace(전체 사용)
- 네임스페이스 안의 **모든 식별자**를 현재 범위로 가져옴
- 코드가 간결해지지만 **이름 충돌 위험** 존재
```
#include <iostream>

using namespace std;

int main() {
    int x;
    cin >> x;
    cout << x << endl;
}
```

### 7. namespace 별칭
#### 별칭이란?
- 긴 네임스페이스 이름을 **짧은 이름**으로 대체하여 사용
```
namespace LongNamespaceName {
    int value = 10;
}

namespace LN = LongNamespaceName;

int main() {
    LN::value;
}
```

### 8. 전역 namespace와 전역 변수 접근
#### 전역 네임스페이스란?
- 어떤 네임스페이스에도 속하지 않은 영역
- `::` 만 사용하여 접근
```
int x = 100;

    int x = 10;

    void func() {
        cout << x << endl;     // A::x
        cout << ::x << endl;  // 전역 x
    }
}
```
- 범위지정 연산자(`{}`)는 지역변수가 아닌 전역변수의 접근에도 사용이 가능

### 1. C 언어에 추가된 기능
C++은 C 언어를 기반으로 여러 기능을 확장한 언어이다.

### 2. C++의 객체지향 특성

#### 객체와 캡슐화 (Encapsulation)
- **캡슐화**
  - 데이터를 하나의 캡슐로 감싸
  - 외부로부터 직접적인 접근을 제한하는 특성
- 이 캡슐의 역할을 하는 것이 **클래스(Class)**

- 클래스
  - 멤버 변수 + 멤버 함수로 구성
  - 접근 지정자(`public`, `private`, `protected`)로 공개 범위 설정 가능
- 클래스에서 생성된 실체를
  - **인스턴스(객체)** 라고 함

- 일반적인 설계 원칙
  - 멤버 변수는 **private**
  - 일부 멤버 함수만 **public**
  - 외부에서는 **간접 접근**만 허용

---

#### 상속성(Inheritance)
- 자식이 부모의 특성을 물려받는 것과 유사한 개념
- C++에서는
  - 클래스 간에 상속 관계를 설정
  - 자식 클래스 객체 생성 시
    - 부모 클래스의 멤버 + 자식 클래스의 멤버를 함께 가짐
- 상속의 장점
  - 코드 재사용성 증가
  - 생산성 향상

---

#### 다형성 (Polymorphism)
- 하나의 기능이
  - 상황에 따라 서로 **다르게 동작하거나**
  - **다르게 보이는** 현상
- 대표적인 구현 방법
  - 함수 오버로딩
  - 함수 오버라이딩

---

### 3. 오버로딩과 오버라이딩

### [문제2] 실행중인 프로그램의 메모리 공간
---
실행중인 프로그램은 운영체제로부터 메모리 공간을 할당 받는데, 이는 크게 데이터, 스택, 힙 영역으로 나뉜다.<br>

| 메모리 영역 | 할당되는 변수/데이터 | 할당 시점 | 소멸 시점 | 특징 |
| :--- | :--- | :--- | :--- | :--- |
| **데이터 (Data)** | **전역변수**, 정적변수(static) | 프로그램 시작 시 | 프로그램 종료 시 | 프로그램 실행 내내 유지되어야 하는 데이터 |

- 스택 영역의 변수와 달리, 함수가 종료되어도 메모리 공간이 자동으로 사라지지 않습니다.

> **free (Memory Release):**
- `malloc`을 통해 할당된 메모리는 반드시 `free` 함수를 호출하여 수동으로 해제

## ch02-3 참조자(reference)의 이해
### 1. 참조자란?
---
- 이미 선언된 변수에 또 다른 이름을 부여하는 것
- **하나의 메모리 공간에 이름만 두개**가 되는 셈(변수명과 사실상 차이가X)
> 연산자 : `&`를 사용<br>
> ex) `홍길동`의 별명을 `쾌도`로 지어주는 것과 같다

### 2. 참조자의 선언과 규칙
---

- 반드시 **선언과 동시에 초기화** 해야한다
- 상수(리터럴)는 참조 불가(변수만 가능)
- 한번 참조하면 대상을 바꿀수 XX
- NULL로 초기화 불가
- 참조자의 수엔 제한이 없으며, 참조자를 대상으로 참조자 선언도 가능

### 3. 참조자의 선언 가능 범위 : [소스](./basic/base/reference/reference.cpp)
---

> int &ref = 20;
- 상수 대상으로의 참조자 선언은 불가능

> int &ref;
- 참조 대상이 없음, 생성과 동시에 참조해야 함

> int &ref = NULL;
- 포인터 처럼 NULL로 초기화하는 것도 불가능

> int &ref1=arr[0];
- **변수의 성향**을 지니는 대상이라면 참조 가능
- **배열의 요소** 역시 그러한 성향을 지니기에 선언 가능
- arr 0번방이라는 이름이 있음

### 4. 포인터 변수 대상의 참조자 선언
```
int main(){
    int num=12;
    int *ptr = &num;
    int **dptr = &ptr;
    
    int &ref = num;
    int* (&pref) = ptr;     // 포인터 ptr의 별명 pref 생성
    int** (&dpref) = dptr;  // 포인터 dptr의 별명 dpref 생성

    cout << ref << endl;
    cout << *pref << endl;
    cout << **dpref << endl;
    // 모두 12를 출력
}
```

| 변수 | 자료형 | 참조자 선언 | 비고 |
| :--- | :--- | :--- | :--- |
| **num** | `int` | `int &ref = num;` | 일반 변수 참조 |
| **ptr** | `int*` | `int* (&pref) = ptr;` | 1중 포인터 변수 참조 |
| **dptr** | `int**` | `int** (&dpref) = dptr;` | 2중 포인터 변수 참조 |

#### 1) 대입 연산자 기준 자료형 일치
참조하려는 대상의 자료형을 그대로 가져와야 함
- `int` 참조 → `int &`
- `int*` 참조 → `int* &`
- `int**` 참조 → `int** &`

#### 2) 포인터도 변수
- `ptr`과 `dptr` 역시 메모리에 주소값을 저장하는 엄연한변수
- 따라서 일반 변수처럼 별명(참조자)을 가질 수 있음

#### 3) 간접 참조 연산자 (*)
- `ref`는 `num` 그 자체이므로 그냥 출력
- `pref`는 `ptr`의 별명이므로, `*pref`는 결국 `*ptr`과 같아 `num`의 값에 접근
- `dpref`는 `dptr`의 별명이므로, `**dpref`는 `**dptr`과 같아 최종적으로 `num`의 값에 접근

### 5. 참조자를 사용하는 이유
#### 1) 거대한 데이터의 효율적 전달
- value로 호출할 경우 통째로 복사
- 그러나 ref로 호출한다면 주소만 전달 -> 복사비용 ~= 0

#### 2) 함수 밖의 값을 직접적으로 수정
- 기본적으로 함수는 하나의 return만 가능
- 참조자를 쓰면 여러개의 값을 수정할 수 있음

#### 3) 코드 블럭 안에서 사용하는경우
> 이름이 너무 길때
- ex) `veryLongGameCharacterName.stats.hp` >> `hp`
> 복잡한 구조체/객체 접근
- 계층이 깊은 데이터를 다룰때 깔끔하게 만들기 위해

> 포인터를 더 편하게 쓰고 싶을때
- 포인터는 `*`이나 `->`를 써야하지만, 참조자는 일반 변수처럼 사용가능

| **디폴트 매개변수** | 기본값(Default Value) 설정 | - 반드시 오른쪽 매개변수부터 설정해야 함<br>- 호출 시 인자는 왼쪽부터 채워짐 |
| **인라인 함수** | 호출 위치에 코드를 직접 치환 | - 호출 오버헤드 제거로 실행 속도 향상<br>- 짧고 빈번히 호출되는 함수에 권장 |

## 3. 매크로 vs 인라인 vs 템플릿
자료형에 독립적인 코드를 작성하기 위한 세 가지 방법
- **매크로(`#define`)**: 전처리기가 단순 치환. 빠르지만 타입 체크가 안 되어 위험함.

- **인라인(`inline`)**: 컴파일러가 처리. 타입 체크가 가능하여 안전함.

- **템플릿(`template`)**: 자료형에 관계없이 컴파일 시점에 함수를 자동 생성. **현대 C++의 핵심.**

## 4. 메모리 구조와 const(C언어 복습)
프로그램이 사용하는 메모리는 용도에 따라 세 부분으로 나뉨
1. **데이터(Data)**: 전역/정적 변수. 프로그램 종료 시까지 유지.

2. **스택(Stack)**: 지역/매개 변수. 함수 종료 시 자동 소멸.

3. **힙(Heap)**: `malloc`/`free`(`new`/`delete`). 개발자가 직접 관리하는 영역.

- `const`는 자신의 바로 오른쪽 대상을 상수로 만듦

    > `const int * ptr` : 가리키는 내용물이 상수

    > `int * const ptr` : 포인터(주소값) 자체가 상수

## 5. 참조자(Reference)
참조자는 변수에 붙이는 **'별명'**이며, 내부적으로는 포인터와 유사하지만 사용법은 변수와 같음

### 참조자의 4가지 규칙
> `선언과 동시에 초기화` 필수.<br>
`상수(리터럴)` 참조 불가.<br>
한 번 정한 대상은 `변경 불가`.<br>
`NULL` 허용 안 함.

### 포인터 변수의 참조자
포인터도 메모리에 존재하는 변수이므로 별명을 붙일 수 있다. **"자료형을 그대로 복사하고 &만 추가한다"**는 원칙을 기억
```
int *ptr = &num;
int* (&pref) = ptr; // ptr(int*)에 대한 참조자(pref)
```

| 대상 | 대상 자료형 | 참조자 선언 |
| :--- | :--- | :--- |
| **일반 변수** | `int` | `int &ref = num;` |
| **1중 포인터** | `int*` | `int* &pref = ptr;` |
| **2중 포인터** | `int**` | `int** &dpref = dptr;` |

## 6. 객체 지향의 요소
#### 캡슐화(Encapsulation): 클래스를 통해 데이터와 기능을 묶고, `private`으로 보호.

#### 상속(Inheritance): 부모의 기능을 물려받아 재사용.

#### 다형성(Polymorphism): 오버로딩/오버라이딩을 통해 하나의 인터페이스로 다양한 동작 수행.

### 8. 객체 생성 문법 : `()` vs `=`

> 이를 **복사 초기화**라고 함
- 최근의 컴파일러는 똑똑해져서 거의 똑같이 처리해주기도 함

### 6. 생성자 심화 : [초기화 리스트와 오버로딩](./basic/day03/init02/init02.cpp)
#### 생성자 오버로딩
매개변수의 개수나 타입에 따라 여려 개의 생성자를 둘 수 있음
- **기본 생성자(`Time()`)** : 인자 없이 `Time now;`라고 선언할 때 호출
- 매개변수가 있는 생성자(`Time(int g, int m, int s)`) : 객체 생성과 동시에 특정 값으로 초기화

#### 콜론 초기화(Member Initializer List)
C++에서 가장 권장되는 멤버 변수 초기화 방식
```
// 대입 연산이 아니라 '초기화' 그 자체를 수행
Time(int h, int m, int s) : m_hour(h), m_min(m), m_sec(s) { }
```
#### 왜쓰는가?
- 생성자 몸체( `{ }` )에서 값을 대입하는 것보다 **성능상 이점**이 있고, 특히 `const`멤버 변수나 `참조자`멤버는 오직 이 방식으로만 초기화가 가능하기 때문

#### 멤버 변수 명명 관례 (`m_`)
- 멤버 변수 앞에 **`m_`**을 붙이는 것은 이 변수가 클래스의 **Member(필드)**임을 명시하는 관례
- **장점** : 매개변수(`h`, `m`, `s`)와 멤버 변수가 섞일 염려가 없고 코드의 가독성을 높여줌

### 7. 초기화 리스트(Initializer List)가 필수인 경우 : [소스](./basic/day03/init03/init03.cpp)
일반적인 멤버 변수는 생성자 몸체(`{ }`) 안에서 대입 연산자로 값을 넣을 수 있지만, 아래의 두 경우에는 반드시 **콜론(:) 초기화 리스트**를 사용
#### 1) 상수 멤버 변수(const)
- 상수는 탄생과 동시에 값이 결정되어야 하며, 이후에는 값 변경(대입) 불가
- 생성자 몸체 안에서 `value = i;`라고 쓰는 것은 '대입'이지 '초기화'가 아니므로 에러가 발생

#### 2) 참조자 멤버 변수(&)
- 참조자는 선언과 동시에 누구를 참조할 지 별명이 붙어야 함
- 참조자 역시 한 번 결정되면 참조 대상을 바꿀 수 없으므로, 생성자 몸체가 시작되기 전에 초기화 리스트에서 연결
